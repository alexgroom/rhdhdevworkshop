        pos = masterUrl.indexOf(".apps.");

        {
            return masterUrl;
        }
        else
        {
            // delivers a URL fragment of the form apps.xxx.yyy.com
            return masterUrl.substring(pos+1);
        }
    }

[#architecture]
== Workshop Architecture
The diagram shows all the components that have integrated with Developer Hub for this lab. Everything today is running in an OpenShift cluster.
We have integrated Keycloak for authentication, GitLab as the Git repo, we use ArgoCD to orchestrate using GitOps from Git into the cluster. We use Tekton 
pipelines to build applications and webhooks to notify the pipelines when Git changes. Finally we use Dev Spaces as the code IDE to view and change the source code.

[#login]
== Login to Developer Hub
Login in to Developer Hub using this link: *link:https://backstage-backstage.%ROUTE_SUBDOMAIN%[Developer Hub^, role='params-link']* 

We have configured KeyCloak to use GitLab as our identity provider. 

You will be redirected to a **KeyCloak** page for authentication.  

image::backstage-auth.png[login, 500]

Click on the **GitLab** button at the *bottom* of the login screen. 
You will now be redirected to the **GitLab** login page.  

image::gitlab-auth.png[gitlab, 500]

Use the credentials to login:
[source,shell,subs="{markup-in-source}",role=copypaste]
----
user1
----
and the password: 
[source,shell,subs="{markup-in-source}",role=copypaste]
----
%ADMIN_PASSWORD%
----

Click *Authorize* to join Keycloak and GitLab.

image::auth-keycloak.png[keycloak, 500]

Now *KeyCloak* will redirect to the Developer Hub (RHDH) home screen. 

image::rhdh-front-page.png[rhdh, 800]

On first use Developer Hub initially displays with a *Quick start* side panel to the right of the screen, you can hide this using the *Hide* 
link in the bottom right of the display, this should allow you to see more information. You can toggle this back using the (?) help button.
